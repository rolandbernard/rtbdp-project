* Groups:
  * one student -> default & typical situation, or
  * ~~two students -> project proposal must justify need for two people~~
* Deliverables:
  * project proposal -> not evaluated, serves to check feasibility & to give you feedback
  * source code -> evaluated, runnable
  * written report -> evaluated, 10-15 pages (∼ 3000 words)
  * slides -> evaluated, 15 min. talk including demo, clear & to the point
* Deadlines (T = chosen exam date):
  * T - 25 days -> 1 page project proposal (hard deadline, better to submit proposal earlier)
  * T - 7 days →-> report & code submission (hard deadline, may optionally include slides)
  * T -> slides submission (if not done) & project presentation: 15" talk incl. demo + 5-15" questions
